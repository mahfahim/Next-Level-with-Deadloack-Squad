# Summary

### ১. Prisma = Translator : ts / js -> sql

Prisma একটি অনুবাদক বা ট্রান্সলেটর হিসেবে কাজ করে। আমরা যখন আমাদের অ্যাপ্লিকেশনে TypeScript (ts) বা JavaScript (js) দিয়ে কোড লিখি, Prisma সেই কোডকে SQL (Structured Query Language) কোডে রূপান্তর করে দেয়, যাতে ডেটাবেস তা সহজে বুঝতে পারে।

### ২. Prisma is an ORM

Prisma হলো একটি **ORM** (Object-Relational Mapping) টুল। এর সুবিধা হলো, ডেটাবেসের টেবিল ও ডেটাগুলোকে অবজেক্ট হিসেবে হ্যান্ডেল করা যায়। ফলে সরাসরি জটিল SQL কুয়েরি না লিখেও ব্যাকএন্ড কোড দিয়েই ডেটাবেসের সব কাজ করা সম্ভব হয়।

### ৩. Prisma Ecosystem

এর অর্থ হলো Prisma-এর পুরো পরিবেশ বা ইকোসিস্টেম। একটি অ্যাপ্লিকেশনের ডেটাবেস ম্যানেজমেন্ট সহজ করার জন্য Prisma যেসব টুলস এবং ফিচার অফার করে, সেগুলোর সমষ্টিই হলো এর ইকোসিস্টেম।

### ৪. Prisma Core Tools (প্রধান টুলসমূহ)

Prisma-এর মূল টুল বা উপাদান তিনটি:

* **Prisma Client:** এটি একটি অটো-জেনারেটেড এবং টাইপ-সেফ কুয়েরি বিল্ডার, যা দিয়ে অ্যাপ্লিকেশনের কোড থেকে ডেটাবেসে ডেটা আদান-প্রদান করা হয়।
* **Prisma Migrate:** এটি ডেটাবেসের স্ট্রাকচার বা স্কিমা তৈরি এবং পরিবর্তন (Migration) করার জন্য ব্যবহৃত হয়।
* **Prisma Studio:** এটি একটি ভিজ্যুয়াল বা গ্রাফিক্যাল ইউজার ইন্টারফেস (GUI), যার মাধ্যমে ব্রাউজারে বসেই ডেটাবেসের ডেটা সরাসরি দেখা, যোগ করা বা এডিট করা যায়।

### ৫. Prisma Works Under The Hood (ভেতরের কাজের প্রক্রিয়া)

Prisma পর্দার আড়ালে যেভাবে ধাপে ধাপে কাজ করে, তা এখানে দেখানো হয়েছে:

> Prisma Migrate → DB Creation → Prisma Client Generate → DB query in TS → Query Engine → Translate

* প্রথমে **Prisma Migrate**-এর মাধ্যমে ডেটাবেস তৈরি বা আপডেট হয় (**DB Creation**)।
* এরপর **Prisma Client Generate** হয়।
* ডেভেলাপার যখন TypeScript-এ কোনো কুয়েরি লেখে (**DB query in TS**), তখন সেটি Prisma-এর **Query Engine**-এ যায়।
* সবশেষে কুয়েরি ইঞ্জিন সেই কোডটিকে SQL-এ অনুবাদ (**Translate**) করে ডেটাবেসে পাঠায়।

### ৬. Prisma Common Terms (সাধারণ কিছু টার্মস)

Prisma ব্যবহার করার সময় প্রায়ই যে শব্দগুলোর মুখোমুখি হতে হয়:

* **Schema & Model:** যেখানে ডেটাবেসের স্ট্রাকচার, টেবিল এবং তাদের মধ্যকার রিলেশন ডিফাইন করা হয়।
* **Datasource & Client Generator:** ডেটাবেস কানেকশন (যেমন: PostgreSQL, MySQL) এবং ক্লায়েন্ট জেনারেট করার সেটিংস।
* **Prisma Client Query:** ডেটাবেস থেকে ডেটা রিড বা রাইট করার জন্য কোড লেখার পদ্ধতি।
* **Prisma Studio DB GUI:** কোড ছাড়া সরাসরি গ্রাফিক্সের মাধ্যমে ডেটাবেস দেখার টুল।
* **Migrate & Generate:** স্কিমার পরিবর্তনগুলো ডেটাবেসে অ্যাপ্লাই করা (Migrate) এবং ক্লায়েন্ট কোড আপডেট করা (Generate)।

### ৭. Prisma Documentation Exploration

এর মানে হলো Prisma-এর অফিশিয়াল ডকুমেন্টেশন বা গাইডলাইন কীভাবে ব্যবহার করতে হয় এবং সেখান থেকে কীভাবে প্রয়োজনীয় তথ্য খুঁজে বের করতে হয়, তা জানা।
